Overview

moju is a JAX-native library for physics-informed and physics-supervised SciML. The package provides moju.piratio (Groups, Models, Laws, Operators) and moju.monitor (ResidualEngine, build_loss, audit, visualize) so you can train or constrain models with the equations you trust.

Installation

pip install moju

Optional: pip install moju[units] for Pint-based units; pip install moju[dev] for pytest, black, ruff.

Quick start

import moju
from moju.piratio import Groups, Models

# Reynolds number and air density
Re = Groups.re(u=1.0, L=0.1, rho=1000.0, mu=1e-3)
rho = Models.ideal_gas_rho(P=101325.0, R=287.0, T=300.0)

API at a glance

  • Groups — Dimensionless numbers: Re, Pr, Nu, Ma, Gr, Pe, St, Bi, Fo, Sc, Le, Kn, Bo, Ca, Eu, Da, Ec.
  • Models — Viscosity (Sutherland, VFT, power-law), density (ideal gas, Boussinesq), heat (Stefan-Boltzmann, Fourier), friction (Darcy-Weisbach, Colebrook), and more.
  • Laws — Residuals for mass, momentum (Navier-Stokes, Stokes, Euler), heat/diffusion, Darcy/Brinkman, Poisson, wave, Burgers, Schrödinger, etc.
  • Operators — gradient, jacobian, divergence, laplacian, curl (2D/3D), time_derivative, time_derivative_second, advection, symmetric_gradient.
  • ResidualEngine (moju.monitor) — compute_residuals, build_loss, audit, visualize; single log for RMS and metrics.

All functions are JIT-compiled and support single-point or batched inputs (leading batch dimension). Use them inside loss functions and training loops with full JAX autodiff.

High-level architecture

User-defined config (Laws, Groups, Models, Constants) and inputs (state_pred, optional state_ref and key_ref) feed into ResidualEngine, which computes residuals and optionally logs per-key RMS. The residual dict drives build_loss (physics-only) for training; the same log is used by audit and visualize for monitoring.

Training and monitoring (ResidualEngine)

ResidualEngine is the single entry point for residuals, physics loss, and monitoring. Import from moju.monitor: from moju.monitor import ResidualEngine, build_loss, audit, visualize. You configure laws, groups, and models; then one class computes residuals, builds a physics-only loss, and keeps a log for audit and visualization.

  • compute_residuals(state_pred, state_ref=None, key_ref=None) — Returns a residual dict: laws always; groups and models when key_ref or state_ref is provided; data residual only when state_ref is provided. Logs per-key RMS to a single log object.
  • build_loss — Physics-only (cascaded over laws). No data residual or loss passed in; you add data loss in JAX or PyTorch to the returned scalar.
  • audit — Reads the logged RMS, computes R_norm, admissibility score, and overall admissibility score; assigns one of four levels per key and overall: 0.00–<0.40 Non-Admissible, 0.40–<0.70 Low Admissibility, 0.70–<0.90 Moderate Admissibility, 0.90–1.00 High Admissibility. Writes these back into the same log. Includes data residual metrics only when state_ref was provided. Use admissibility_level(score) for the same mapping. Optional PDF export: pass export_dir (and optionally save_residuals=True, last_residual_dict=...) to write a Physical Admissibility Report PDF and a zipped results folder; requires pip install moju[report].
  • visualize — Plots RMS per key and admissibility score per key as training proceeds and/or at the end.

key_ref is for Groups and Models only (reference values for group/model outputs). Data residual is computed and logged only when state_ref is provided.

Custom physics (optional fn)

You can use your own JAX-differentiable functions as laws, groups, or models instead of (or alongside) the built-in ones. In any spec, add an optional "fn": your_callable. The engine will call your function with keyword arguments derived from state_map (same as for built-in names). Your function must accept those kwargs and return JAX arrays so the pipeline stays differentiable.

Laws. Spec: {"name": "my_law", "state_map": {"x": "state_key"}, "fn": my_residual_fn}. my_residual_fn(x=...) should return the residual (e.g. R = 0 when satisfied). The residual is used in build_loss and in the log.

Groups and models. Spec: {"name": "my_group", "state_map": {"a": "a", "b": "b"}, "output_key": "ab", "fn": my_fn}. Your function is called with kwargs from state_map; its return value is written to output_key in the state and (when key_ref is provided) used for the group/model residual.

Use JAX/NumPy (jax.numpy) inside your fn so it remains differentiable. Example: custom law

import jax.numpy as jnp
from moju.monitor import ResidualEngine, build_loss

def my_residual(x):
    return x - 1.0  # residual = 0 when x == 1

engine = ResidualEngine(
    constants={},
    laws=[{"name": "my_law", "state_map": {"x": "x"}, "fn": my_residual}],
    groups=[], models=[],
)
state = {"x": jnp.array(2.0)}
residuals = engine.compute_residuals(state)
# residuals["laws"]["my_law"] == 1.0
loss = build_loss(residuals)  # physics loss from your custom law
