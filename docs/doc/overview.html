Overview

moju is a JAX-native library for physics-informed and physics-supervised SciML. The package provides moju.piratio (Groups, Models, Laws, Operators) and moju.monitor (ResidualEngine, build_loss, audit, visualize) so you can train or constrain models with the equations you trust.

Installation

pip install moju

Optional: pip install moju[units] for Pint-based units; pip install moju[dev] for pytest, black, ruff; pip install moju[ref] for xarray-based state_ref loaders (CFD snapshots → dict with optional interpolation); pip install moju[ref_vtk] for VTU/VTK loaders; pip install moju[ref_foam] for OpenFOAM loaders; pip install moju[ref_hdf5] for HDF5 loaders.

Quick start

import moju
from moju.piratio import Groups, Models

# Reynolds number and air density
Re = Groups.re(u=1.0, L=0.1, rho=1000.0, mu=1e-3)
rho = Models.ideal_gas_rho(P=101325.0, R=287.0, T=300.0)

API at a glance

  • Groups — Dimensionless numbers: Re, Pr, Nu, Ma, Gr, Pe, St, Bi, Fo, Sc, Le, Kn, Bo, Ca, Eu, Da, Ec.
  • Models — Viscosity (Sutherland, VFT, power-law), density (ideal gas, Boussinesq), heat (Stefan-Boltzmann, Fourier), friction (Darcy-Weisbach, Colebrook), and more.
  • Laws — Residuals for mass, momentum (Navier-Stokes, Stokes, Euler), heat/diffusion, Darcy/Brinkman, Poisson, wave, Burgers, Schrödinger, etc.
  • Operators — gradient, jacobian, divergence, laplacian, curl (2D/3D), time_derivative, time_derivative_second, advection, symmetric_gradient.
  • ResidualEngine (moju.monitor) — compute_residuals, build_loss, audit, visualize; single log for RMS and metrics.

All functions are JIT-compiled and support single-point or batched inputs (leading batch dimension). Use them inside loss functions and training loops with full JAX autodiff.

High-level architecture

User config (Laws, Groups, optional constitutive_audit / scaling_audit, Constants) and inputs (state_pred with predicted fields, optional state_ref for data residual) feed into ResidualEngine. Merged state = state_pred + constants + group outputs (output_key). Residuals: laws, constitutive and scaling closure checks, and data when state_ref is set. build_loss uses laws only; audit / visualize use the full log.

Training and monitoring (ResidualEngine)

ResidualEngine: from moju.monitor import ResidualEngine, MonitorConfig, AuditSpec, build_loss, audit, visualize. Configure laws, groups, and optional constitutive_audit / scaling_audit audit specs tied to Models.* and Groups.* functions. Prefer the typed config (MonitorConfig, AuditSpec) for IDE autocompletion; serialize with to_dict()/from_dict(). Two entry paths are supported: Path A (recommended) builds state_pred via a state_builder from (model, params, collocation); Path B accepts state_pred directly (advanced, user supplies derivative keys). Closure keys are logged under constitutive/<name>/... and scaling/<name>/... (e.g. chain_dx, chain_dt, ref_delta).

  • compute_residuals(...) — Path B: compute_residuals(state_pred, state_ref=None, ...). Path A: compute_residuals(state_pred=None, model=..., params=..., collocation=..., ...) with ResidualEngine(state_builder=...). Always runs laws. For Models/Groups audits, moju omits items with no spatial and no temporal variation; otherwise includes only applicable closures (chain_dx, chain_dt, and ref_delta when state_ref is provided). Use required_state_keys() / required_derivative_keys() to see required keys. Logs per-key RMS. Use log_to_python=False inside jax.jit / jax.grad.
  • build_loss — Cascaded over laws only.
  • audit — R_norm(k) = RMS(r_k)/scale_k; scale is state-derived by default (stored in each log entry as entry["scale"]). Passing r_ref overrides scale for those keys (e.g. baseline or training curves). Admissibility levels, optional PDF (pip install moju[report]). Report groups metrics under Governing laws, Constitutive relations, Scaling and similarity, and Data when applicable.
  • visualize — RMS and admissibility per logged key.

Discover valid AuditSpec.name ids: AuditSpec.name must match the registry function id (e.g. sutherland_mu for Models.*, pe for Groups.*).

from moju.monitor import list_constitutive_models, list_scaling_closure_ids

print("Constitutive (Models.*) ids:", list_constitutive_models())
print("Scaling (Groups.*) ids:", list_scaling_closure_ids())

Derivative strategies (Path B / CFD): for chain_dx/chain_dt, provide derivative keys (e.g. d_T_dx, d_mu_dt) using solver gradients when available; otherwise use coordinate-aware finite differences (and optionally smoothing before differencing). Planned extension point: a weak-form mode that integrates closure residuals over space/time windows via quadrature (API hook: closure_mode="pointwise"|"weak" plus quadrature_weights).

CFD snapshot cookbook: see examples/cfd_snapshot_cookbook_heat_1d.py for an end-to-end reference workflow (xarray snapshot → optional interpolation → finite-difference gradients + smoothing → weak-form audit → interpret RMS/admissibility).

Custom physics (optional fn)

In law or group specs, add "fn": your_callable; kwargs from state_map. Laws return the PDE residual. Groups return a value stored at output_key in merged state.

For extra closure residuals, use constitutive_custom / scaling_custom or register-style lists on the engine.

import jax.numpy as jnp
from moju.monitor import ResidualEngine, build_loss

def my_residual(x):
    return x - 1.0

engine = ResidualEngine(
    constants={},
    laws=[{"name": "my_law", "state_map": {"x": "x"}, "fn": my_residual}],
    groups=[],
)
state = {"x": jnp.array(2.0)}
residuals = engine.compute_residuals(state)
loss = build_loss(residuals)
